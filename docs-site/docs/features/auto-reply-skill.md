# 자동 답장 스킬

[보낸 메일함 페르소나 분석](./sent-mail-persona-analysis.md)에서 "반복적이고 어렵지 않은" 것으로 분류된 답장 패턴은, 실제로 실행 가능한 **agent skill**로 변환됩니다. 이 스킬을 통해 WorkTwin은 유사한 메일이 다시 들어왔을 때 나를 대신해 답장할 수 있습니다.

## 패턴에서 스킬로

하나의 답장 패턴이 스킬이 되려면 최소한 다음 요소가 정의되어야 합니다.

- **트리거 조건**: 어떤 유형의 수신 메일이 이 스킬의 대상인지 (발신자, 주제 키워드, 스레드 맥락 등)
- **판단 로직**: 과거 답장에서 반복적으로 나타난 조건-결과 규칙 (예: "요청 기한이 2주 이상 남아 있으면 승인, 그렇지 않으면 일정 조율 제안")
- **응답 템플릿/스타일**: 실제 문면을 만들 때 사용할 어조와 구조 — 이는 [페르소나 분석](./sent-mail-persona-analysis.md)에서 추출된 스타일을 그대로 재사용합니다.
- **신뢰 범위**: 이 스킬이 사람의 확인 없이 바로 발송해도 되는지, 아니면 초안만 만들고 사람이 검토 후 보내야 하는지

## 실행 모드

자동화의 신뢰 수준에 따라 최소 두 단계의 실행 모드를 구분할 수 있습니다.

1. **초안 제안 모드**: WorkTwin이 답장 초안을 작성해 사용자에게 보여주고, 사용자가 확인 후 발송합니다. 자동화 초기 단계에서 기본이 되는 모드입니다.
2. **자동 발송 모드**: 신뢰도가 충분히 검증된 스킬에 한해, 사람의 개입 없이 바로 답장을 발송합니다.

:::warning 신중해야 할 지점
자동 발송 모드는 잘못된 답장이 실제로 상대방에게 전달된다는 리스크를 가집니다. 어떤 기준으로 "이 스킬은 자동 발송해도 안전하다"를 판단할지, 그리고 실수가 발생했을 때 되돌릴 수 있는 장치(예: 발송 전 유예 시간, 발송 로그와 되돌리기)를 어떻게 둘지는 신중히 설계해야 할 부분입니다.
:::

## 스킬은 계속 누적된다

새로운 유형의 반복 답장이 발견될 때마다 스킬 목록에 추가되는 구조를 지향합니다. 즉 자동 답장 기능은 "완성된 기능"이 아니라, **동기화가 계속될수록 커버리지가 넓어지는 성장형 기능**입니다. 이 스킬들은 이후 [역할별 서브에이전트](./role-extraction-subagents.md)에 소속되어, 어떤 역할을 수행할 때 어떤 스킬 세트를 쓸지 구분됩니다.
