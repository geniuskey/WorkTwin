# 보낸 메일함 페르소나 분석

받은 메일함이 "내가 무엇을 알고 있는가"를 담고 있다면, **보낸 메일함은 "내가 어떻게 판단하고 반응하는가"를 담고 있습니다.** WorkTwin은 보낸 메일함을 별도로 분석해 나의 업무 페르소나를 추출합니다.

## 무엇을 분석하는가

보낸 메일 하나하나는 단순한 텍스트가 아니라, 다음과 같은 **판단의 흔적**입니다.

- 어떤 요청에 **승인/반려/보류** 중 무엇으로 답했는가
- 같은 유형의 질문에 **얼마나 일관된 논리**로 답했는가
- 어떤 어조(격식/캐주얼, 단답/상세 설명)를 상황별로 구분해서 썼는가
- 리뷰 메일에서 **어떤 기준**(품질, 일정, 리스크 등)을 우선시했는가
- 어떤 요청에는 답장을 미루거나 다른 사람에게 위임했는가

이런 패턴은 원문 몇 통을 읽어서는 보이지 않지만, 수백~수천 통을 유형별로 묶어 비교하면 뚜렷한 규칙성으로 드러납니다.

## 분석 결과물

보낸 메일함 분석의 결과물은 크게 두 갈래로 이어집니다.

1. **답장 패턴 → 자동화 대상**: 어렵지 않고 반복적인 유형의 답장은 패턴화해서 [자동 답장 스킬](./auto-reply-skill.md)로 만듭니다.
2. **역할과 판단 기준 → 구조화 대상**: 어떤 맥락에서 어떤 기준으로 판단했는지는 [역할 추출과 서브에이전트](./role-extraction-subagents.md)를 정의하는 근거 데이터가 됩니다.

즉 이 분석 자체가 최종 산출물이 아니라, WorkTwin의 나머지 핵심 기능들이 공유하는 **공통 입력 데이터**라고 보는 것이 정확합니다.

## "어렵지 않은 일"을 가려내는 문제

모든 답장이 자동화 대상이 될 수는 없습니다. 예를 들어 일정 조율이나 정형화된 상태 확인 요청은 패턴이 뚜렷하지만, 이해관계가 걸린 의사결정이나 처음 보는 유형의 문의는 그렇지 않습니다. 따라서 페르소나 분석 단계에서는 각 답장 패턴에 대해 **반복성**과 **판단 난이도**를 함께 평가해, 이후 자동화 여부를 가를 수 있는 신호를 남겨야 합니다. 이 기준을 정확히 어떻게 수치화할지는 아직 열려 있는 질문입니다 ([로드맵](../roadmap.md) 참고).
