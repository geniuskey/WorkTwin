# 핵심 개념: WorkTwin

## 한 문장 정의

> **WorkTwin은 내 메일함(받은 편지함 + 보낸 편지함) 전체를 학습해, 나를 대신해 메일 맥락을 검색·이해·응답할 수 있는 개인 전용 에이전트다.**

이름 그대로 "업무상의 나(Work)"를 복제한 "쌍둥이(Twin)"를 지향합니다. 다만 이 쌍둥이는 처음부터 완성된 형태로 존재하지 않고, **메일함이라는 데이터를 통해 점진적으로 나를 학습**해가는 존재입니다.

## 세 가지 축

WorkTwin의 개념은 서로 다른 방향에서 오는 세 가지 능력이 결합된 형태로 이해하면 명확해집니다.

### 1. 아는 것 (Knowing) — 받은 메일함 → RAG

받은 메일함을 분류·정리해서 RAG 벡터 DB에 넣습니다. 이를 통해 "내가 무엇을 알고 있는가"를 챗봇이 대신 검색해줄 수 있습니다. 수 기가바이트짜리 메일함도 대화형으로 탐색이 가능해집니다. → [메일함 동기화·RAG 색인](./features/mailbox-sync-rag.md), [메일함 챗봇 검색](./features/chatbot-inbox-search.md)

### 2. 답하는 것 (Responding) — 보낸 메일함 → 답장 스킬

보낸 메일함을 분석해서 "내가 어떤 기준으로 어떻게 답장/리뷰를 하는지"의 패턴을 찾아냅니다. 어렵지 않은 유형의 답장은 이 패턴을 에이전트 스킬로 만들어 WorkTwin이 나 대신 초안을 쓰거나 바로 응답하게 합니다. → [보낸 메일함 페르소나 분석](./features/sent-mail-persona-analysis.md), [자동 답장 스킬](./features/auto-reply-skill.md)

### 3. 맡은 것 (Being) — 메일함 전체 → 역할과 서브에이전트

메일함 전체를 가로질러 보면 내가 실제로 수행해온 **여러 개의 역할**이 드러납니다. 예를 들어 "프로젝트 리드로서의 나", "코드/문서 리뷰어로서의 나", "벤더와 협상하는 나"는 각기 다른 판단 기준과 어조, 액션 패턴을 가집니다. WorkTwin은 이 역할들을 식별하고, 역할별로 별도의 서브에이전트로 승격시켜 관리합니다. → [역할 추출과 서브에이전트](./features/role-extraction-subagents.md)

## 왜 "나보다 나를 잘 아는"인가

세 축이 쌓이면 WorkTwin은 단순한 검색 도구를 넘어서게 됩니다.

- 나는 3년 전 어떤 프로젝트에서 무슨 결정을 내렸는지 기억하지 못하지만, WorkTwin은 그 메일을 정확히 찾아낼 수 있습니다.
- 나는 내가 맡고 있는 역할들을 명시적으로 나열해본 적이 없지만, WorkTwin은 메일 패턴 분석만으로 그 역할 목록을 만들어낼 수 있습니다.
- 나는 매번 비슷한 답장을 새로 쓰지만, WorkTwin은 이미 그 패턴을 학습해 즉시 적용할 수 있습니다.

즉 WorkTwin이 지향하는 것은 "나를 흉내 내는 챗봇"이 아니라, **내 업무 이력이라는 1차 데이터에서 직접 추출된, 나보다 더 구조화된 형태로 나를 기억하는 에이전트**입니다.

:::note 아직 정의되지 않은 것
"어렵지 않은 일"의 기준, 역할 간 경계, 자동 응답의 신뢰 범위 등은 아직 구체적으로 확정되지 않았습니다. 이는 [로드맵](./roadmap.md)의 열린 질문에서 다룹니다.
:::
